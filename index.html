使用GitHubPages-Hexo创建个人博客

发表于 2023-07-31 分类于 Hexo

环境

  • Node.js = 18.17.0
  • Hexo = 6.3.0

1. 准备工作

Hexo安装教程 GitHubPages

2. 使用Git Bash进行初始化配置

受网络情况影响，可能有的步骤会报错，可以多试几次。建议全程科学上网。

  1. 在你的项目文件夹下根目录，鼠标右键打开Git Bash
  2. 输入hexo init，进行初始化
  3. 输入hexo g，进行页面生成
  4. 输入hexo s，启动本地服务
  5. 打开浏览器，输入 http://localhost:4000/ 即可查看网页预览效果
  6. 按下Ctrl+C，停止服务
  7. 打开项目文件根目录，打开_config.yml文件
  8. 修改最后一段文本
  9. 输入npm install hexo-deployer-git --save，安装插件
  10. 输入hexo g，进行页面生成
  11. 输入hexo d，将网页上传到Github进行托管
    1. 中间需要进行Github登录授权
  12. 进入创建的user_name.github.io储存库
  13. 点击Setting，点击Pages
  14. 在右侧找到Visit Site，即可查看自己的网站（在网址栏中直接输入user_name.github.io也可）

GitHubPages

发表于 2023-07-31 分类于 GitHub

介绍

GitHubPages是一项静态站点托管服务，它直接从GitHub上的仓库获取HTML、CSS和JavaScript文件，通过构建过程运行文件，然后发布网站。

使用

  1. 首先你需要创建一个Github账号
  2. 然后在创建一个名字为user_name.github.io的库
  3. 点击个人头像，进入Setting界面
  4. 点击Emails，将Keep my email addresses private前的勾去掉
  5. 将网站代码上传至仓库，参考：使用GitHubPages-Hexo创建个人博客
  6. 进入创建的user_name.github.io储存库
  7. 点击Setting，点击Pages
  8. 在右侧找到Visit Site，即可查看自己的网站（在网址栏中直接输入user_name.github.io也可）

使用Cloudflare加速GitHubPages

发表于 2023-07-31 分类于 GitHub

本文原作者：Sirius
文章转载于：
https://siriusq.top/github-pages-%E5%90%AF%E7%94%A8-cloudflare-%E5%8A%A0%E9%80%9F%E5%8F%8A-https.html

使用Cloudflare加速

  1. 注册Cloudflare账号
  2. 点击Add zone（右上角可以设置语言）
  3. 输入你的域名，然后点击继续
  4. 选择方案Free（一般够用，有钱可以用别的）
  5. Cloudflare会自动获取域名的DNS记录，这里跳过，点击页面最底下的*继续
  6. 按照Cloudflare给的教程，换掉域名注册商的域名服务器
  7. 回到Cloudflare，点击Done, check nameservers，等待生效
  8. 点击SSL/TLS,概述中，将SSL/TLS加密模式设置为完全；边缘证书中，开启始终使用HTTPS和随机加密；源服务器中，开启经过身份验证的源服务器拉取
  9. 生效后到DNS设置里添加四条 A 记录与一条 CNAME 记录
    • CNAME 记录的 NAME 设置为www，Content 设置为 GitHubPages 默认的域名用户名.github.io
    • A 记录的 Name 都使用@或者自己的域名，Content 依次设置为：
      1. 185.199.108.153
      2. 185.199.109.153
      3. 185.199.110.153
      4. 185.199.111.153
  10. 确认一下各个记录的代理状态都是已代理状况

开启 Encryption Full (strict) 模式

  1. 将Cloudflare中的DNS设置中的记录的代理状态改成仅DNS
  2. 回到GitHubPages，刷新一次，然后尝试能否勾选Enforce HTTPS，如果不行，就等一会再刷新试试
  3. 当在GitHubPages中勾选完Enforce HTTPS后，等其生效
  4. 回到Cloudflare，将代理状态恢复到已代理的状态
  5. 点击SSL/TLS,概述中，将SSL/TLS加密模式设置为完全（严格）；

Git与GitHub仓库进行绑定

发表于 2023-07-30 更新于 2023-07-31 分类于 Tool

1. 设置用户名和邮箱信息

2. 设置本地仓库

  1. 进入到你的本地仓库文件夹，右键打开Git Bash
  2. 输入git init，进行仓库初始化，此时会生成一个 .git 隐藏文件夹
  3. 将你要上传的文件放到你的本地仓库
  4. 输入git add [你要加入暂存区的文件]，git add .指将所有文件加入暂存区
  5. 输入git commit -m "更新文件的注释"，将所有暂存区文件上传本地仓库

3. 设置远程仓库

  1. 右键打开Git Bash，输入ssh-keygen -t rsa -C "你的邮件地址"
  2. 选择默认路径，输入GitHub密码
  3. 在 /C/Users/用户名/.ssh 路径下，找到id_rsa.pub文件，用记事本打开，复制所有内容
  4. 进入Github，点击个人头像，点击Setting，点击SSH and GPG keys
  5. 点击New SSH key，将复制的文本粘贴到Key下边的框内，Title自己起一个
  6. 点击Add SSH key
  7. 点击左侧导航栏中的Emails，将Keep my email addresses private前的勾去掉
  8. 输入ssh -T git@github.com，测试SSH是否绑定成功（中间输入yes）
  9. 点击左上角的Github图标，回到主页
  10. 在左上方有新建储存库按钮，新建远程仓库

4. 进行连接

  1. 在本地仓库文件夹下，右键打开Git Bash
  2. 进入新建的远程仓库，点击Code，点击SSH，复制下方的SSH链接
  3. 输入git remote add origin 复制的SSH链接
  4. 输入git push -u origin main，上传本地仓库
  5. git push <远程主机名> <本地分支名>:<远程分支名>
  6. 刷新Github仓库，看看是否上传成功
